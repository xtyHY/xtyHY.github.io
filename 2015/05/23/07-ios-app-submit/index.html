iOS开发笔记 - iTunes Connection 应用提交

发表于 2015-05-23 | 分类于 iOS开发 |
有了证书、真机测试，项目也写得差不多了，应该要上线了，还需要以下几个步骤，以一个虚构的项目作为案例来提交步骤解释

进入iTunes Connection填写App信息

从苹果开发者中心登陆后进入iTunes Connection，或者直接进入iTunes Connection

新建一个App

新建App

填写相关信息

官方说明：

  1. 名称
    您的 App 在 App Store 中显示的名称。名称长度不能超过 255 个字符。

  2. 主要语言
    如果 App 信息没有针对某个国家的 App Store 进行本地化, App 信息将以主要语言呈现。

  3. 套装ID
    套装 ID 必须与您在 Xcode 中使用的 ID 相符。该 ID 在您上传第一个构建版本后便无法更改。

  4. SUK
    您 App 专有的 ID, 此 ID 不会在 App Store 中显示。

注：名称一般和工程的Display Name一致，套装ID实际就是上篇博文中的证书配置中配置的一系列证书直到最后一步，SUK建议以公司域名反写+发布日期，用于生成唯一AppId

确定价格

完善提交信息

截图上传

官方说明

屏幕快照必须为 JPG 或 PNG 格式，且必须采用 RGB 颜色空间。 App 视频预览必须为 M4V、MP4 或 MOV 格式，且不能超过 500 MB

一般要求

需要四种尺寸的图片，最多能5张，最少1张，视频最多1个（可以没有）

尺寸分别为：3.5英寸（640x960）、4英寸（640x1136）、4.7英寸（750x1334）、5.5英寸（1242x2208）

描述、关键字、支持网址等信息

官方说明

  1. 描述
    对您 App 的描述，用以详细说明特性和功能。它还将用于您的  Watch App。

  2. 关键字
    一个或多个关键词，用以描述您的 App。关键词将使 App Store 搜索结果更加准确。关键词之间用英文逗点分隔。

  3. 支持网址
    您的 App 技术支持信息网址(URL)。该网址将会在 App Store 中显示。

  4. 营销信息网址
    您的 App 营销信息网址(URL)。该网址会在 App Store 中显示。

构建版本

可使用Xcode 或 Application Loader进行构建

图标、版权、联系方式

官方说明

  1. 图标
    此图标将用于 App Store，其格式必须为 JPG 或 PNG，最低分辨率至少为 72 DPI，并采用 RGB 色彩空间。它不能包含图层或圆角。

  2. 版权
    拥有您的 App 专有权的人员或实体的名称，前面是获得权利的年份（例如“2008 Acme Inc”）。请勿提供网址(URL)。

分级

一般来说先全选无，然后再调整，全无4+审核非常严

审核信息

  1. 联系信息
    当 App 审核团队有任何问题或需要额外信息时, 您所在机构的联系人信息。

  2. 备注
    对审核过程会有所帮助的、有关您 App 的额外信息, 包括在测试中需要的 App 特别设置等。

  3. 演示账户
    对您的 App 具有完全访问权限帐户的用户名和密码。此帐户在审核过程中使用，并且不能过期。在“备注”栏中提供其他帐户的详细信息。

注：演示账户就是你的功能里有需要登录的功能，必须提供现有的账户密码给测试，否知会被拒

选择版本发布时间

一般是自动发布，审核一般用时1-3周（不被拒的情况下）

使用Xcode提交

注：需要先配置真机测试的证书（方式见iOS开发笔记 – 证书申请 真机测试配置 - 七、配置真机测试）后，并连接真机，才能进行提交

进行打包

Xcode->Product->Archive

提交到iTunes Connection

选择Submit即可提交到iTunes Connection构建版本

提交审核

选择构建的版本

成功后，刷新iTunes Connection页面，并在构建版本处进行添加（点击+号）

提交审核

最后，iTunesConnect中信息均准备好了之后，点击“提交以供审核”，弹出新页面如下图：

  1. 有3个勾选项，描述均清楚，一般性应用均为勾选“否”

  2. 尤其需要注意的是IDFA，是由你程序中是否用到了IDFA作为唯一ID决定的，请务必根据实际情况进行勾选

最后审核大概时间为1-3周，祝各位App顺利过审
